# Лабораторная работа №9: Интеграция камеры и QR-кода для аутентификации

## Описание задания

Необходимо расширить существующее приложение с авторизацией функционалом работы с камерой и QR-кодами. Реализовать возможность генерации QR-кода с учётными данными пользователя и его последующего сканирования для автоматической авторизации. Добавить звуковое сопровождение и push-уведомления для обратной связи.

## Исходные условия

- База: приложение из лабораторной работы №8 в ветке `task-8`
- Новое решение реализуется в ветке **`task-9`**
- Сохранить всю существующую функциональность приложения

## Основные функции

### 1. Генерация QR-кода

#### На экране профиля пользователя (после авторизации):
- Добавить кнопку **"Создать QR-код авторизации"**
- По нажатию генерировать QR-код, содержащий:
  - Логин пользователя
  - Пароль пользователя
  - Формат данных: JSON или разделённая строка (например: `login:password`)

#### Сохранение изображения:
- После генерации QR-кода отобразить диалог с превью
- Предоставить кнопку **"Сохранить в галерею"**
- При сохранении запрашивать разрешение `WRITE_EXTERNAL_STORAGE` (если API < 30) или использовать `MediaStore` (API >= 30)
- Сохранять изображение в папку приложения в галерее

### 2. Сканирование QR-кода

#### На экране входа:
- Добавить кнопку **"Авторизация через QR-код"** под полями ввода
- По нажатию открывать экран сканирования QR-кода с камерой

#### Экран сканирования:
- Отображать превью с камеры в полноэкранном режиме
- Показывать рамку для наведения на QR-код
- Реализовать таймер обратного отсчёта: **30 секунд**
- По истечении времени автоматически закрывать экран сканирования
- При успешном считывании автоматически вставлять данные в поля ввода на экране авторизации

## Технические требования

### 1. Работа с камерой

#### Разрешения:
- Запрашивать разрешение `CAMERA` через явный диалог перед использованием
- Обрабатывать случай отказа пользователя в разрешении
- Показывать понятное объяснение необходимости разрешения

#### Реализация:
- Использовать **CameraX** API (рекомендуется) или **ZXing**
- Настроить предварительный просмотр (Preview)
- Реализовать анализ изображения (ImageAnalysis) для детекции QR-кодов

Пример запроса разрешений:
```kotlin
val cameraPermissionLauncher = registerForActivityResult(
    ActivityResultContracts.RequestPermission()
) { isGranted ->
    if (isGranted) {
        startCamera()
    } else {
        showPermissionDeniedDialog()
    }
}

// Запуск при необходимости
cameraPermissionLauncher.launch(Manifest.permission.CAMERA)
```

### 2. Генерация и распознавание QR-кодов

#### Генерация:
- Использовать библиотеку **ZXing** или **Android QR Code Generator**
- Создавать Bitmap с QR-кодом заданного размера
- Добавлять отступы (quiet zone) вокруг кода

#### Распознавание:
- Использовать `QRCodeReader` из ZXing или встроенные возможности CameraX
- Обрабатывать результат асинхронно
- Валидировать формат считанных данных

### 3. Звуковое сопровождение

Добавить два звуковых файла в `res/raw/`:
- `qr_success.mp3` — успешное считывание
- `qr_failure.mp3` — неудачное считывание/истечение времени

Реализовать проигрывание:
```kotlin
// Инициализация
private val successSound = MediaPlayer.create(context, R.raw.qr_success)
private val failureSound = MediaPlayer.create(context, R.raw.qr_failure)

// Воспроизведение
successSound.start() // При успешном считывании
failureSound.start() // При ошибке или таймауте
```

### 4. Push-уведомления

Реализовать уведомления через `NotificationManager`:

#### Успешное считывание:
- Заголовок: "Авторизация готова"
- Текст: "Данные из QR-кода загружены. Перейдите к авторизации."
- Иконка: success/check
- Автоматическое исчезновение через 5 секунд

#### Неудачное считывание:
- Заголовок: "Сканирование не удалось"
- Текст: "QR-код не распознан или время истекло."
- Иконка: warning/error
- Длительность: normal

Регистрация канала уведомлений:
```kotlin
val channel = NotificationChannel(
    QR_SCAN_CHANNEL_ID,
    "Сканирование QR-кода",
    NotificationManager.IMPORTANCE_DEFAULT
).apply {
    description = "Уведомления о результатах сканирования QR-кода авторизации"
}

notificationManager.createNotificationChannel(channel)
```
